# 博客操作守则

## 一、上传文章

### 1. 创建文章文件
- 在 `source/_posts` 目录下创建 Markdown 文件
- 文件命名格式：`年-月-日-文章标题.md`（例如：`2026-04-22-测试文章.md`）

### 2. 文章格式
文章开头需要添加 Front Matter 信息，示例如下：
```yaml
---
title: 文章标题
date: 2026-04-22 12:00:00
categories: [分类1, 分类2]
tags: [标签1, 标签2]
description: 文章描述
---

文章内容...
```

### 3. 发布文章
1. 打开命令行，进入博客根目录
2. 执行命令生成静态文件：`npx hexo generate`
3. 执行命令部署到 GitHub Pages：`npx hexo deploy`

## 二、上传图片

### 1. 图片存储位置
- 将图片文件上传到 `source/img` 目录
- 建议按照分类创建子目录，例如 `source/img/posts/` 存放文章图片

### 2. 在文章中引用图片
使用相对路径引用图片，示例：
```markdown
![图片描述](/img/posts/图片文件名.jpg)
```

## 三、更换壁纸

### 1. 更换全局背景
- 编辑 `_config.anzhiyu.yml` 文件
- 找到 `background` 配置项，修改 `url` 值为新的壁纸图片路径
- 示例：
```yaml
background:
  enable: true
  url: /img/background.jpg
  fixed: true
  cover: false
```

### 2. 更换页面特定背景
- 可以在文章的 Front Matter 中添加 `top_img` 字段来设置该文章的顶部背景
- 示例：
```yaml
---
title: 文章标题
date: 2026-04-22 12:00:00
top_img: /img/posts/文章背景.jpg
---
```

## 四、其他操作

### 1. 本地预览
- 执行命令启动本地服务器：`npx hexo server -p 4000`
- 在浏览器中访问：`http://localhost:4000/meloway-station/`

### 2. 清理缓存
- 执行命令清理缓存：`npx hexo clean`
- 然后重新生成静态文件：`npx hexo generate`

### 3. 常用命令
- `npx hexo new "文章标题"`：创建新文章
- `npx hexo clean`：清理缓存
- `npx hexo generate`：生成静态文件
- `npx hexo server`：启动本地服务器
- `npx hexo deploy`：部署到 GitHub Pages

## 五、注意事项

1. 文章文件必须使用 Markdown 格式
2. 图片路径要使用相对路径，以 `/` 开头
3. 部署前请确保已生成静态文件
4. 更换壁纸后需要重新生成静态文件并部署
5. 若遇到问题，可以查看控制台错误信息或检查配置文件
